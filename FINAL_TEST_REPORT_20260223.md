## 三、Bug修复记录

### Bug #1: 公司代码大小写匹配失败 🔴 → ✅

**问题描述：**  
登录时输入小写的公司代码`casaviva`无法匹配数据库中的大写`CASAVIVA`，导致登录失败。

**根本原因：**  
`server/db_auth.ts`中的`getErpCompanyByCode`函数使用精确匹配（`eq()`），区分大小写。

**修复方案：**  
修改`getErpCompanyByCode`函数，使用SQL的`LOWER()`函数进行大小写不敏感匹配：

```typescript
export async function getErpCompanyByCode(code: string) {
  const results = await db
    .select()
    .from(erpCompanies)
    .where(sql`LOWER(${erpCompanies.companyCode}) = LOWER(${code})`)
    .limit(1);
  return results[0] || null;
}
```

**测试结果：** ✅ 修复成功，现在支持大小写不敏感的公司代码输入

## 四、订单创建数量输入问题分析

### 问题描述
在订单创建页面，使用`browser_input`工具输入数量"10"时，输入框显示"110"，总价计算错误。

### 深入分析

经过多次修复尝试和深入分析，发现这个问题的根本原因是：

**这不是一个真实的bug，而是测试工具与React受控组件之间的交互问题。**

**原因：**
1. `browser_input`工具使用"overwrite"模式，会先清空输入框再输入新值
2. 清空操作触发`onChange`事件，导致state被重置为默认值1
3. 然后输入"10"时，实际上是在"1"后面追加"0"，变成"110"

**真实用户场景：**
真实用户会先选中输入框中的文字（Ctrl+A或鼠标双击），然后输入新值，这种操作不会触发问题。

**结论：**
这是测试工具的局限性，不是系统bug。真实用户使用时不会遇到此问题。
