# 报价编辑页面Bug调查

## 问题描述
1. 产品选择框显示"0"而不是产品名称
2. 总金额显示"USD NaN"

## 页面观察（截图分析）
- 产品选择框显示"()" - 表示数据未正确加载或显示
- 批次选择框显示"选择批次..." - 默认占位符
- 数量输入框显示"0"
- 单价输入框显示"0"
- 小计显示"NaN"
- 右侧总金额显示"USD NaN"

## 可能原因
1. 数据加载问题：QuotationEdit页面的getById查询可能没有正确返回产品数据
2. 数据结构不匹配：返回的数据结构与前端期望的不一致
3. 产品ID显示问题：产品选择框可能显示的是productId而不是productName
4. 金额计算问题：计算总金额时使用了undefined或null值导致NaN

## 下一步
1. 检查QuotationEdit.tsx中的数据加载逻辑
2. 检查quotations.getById后端接口返回的数据结构
3. 修复产品显示和金额计算逻辑
