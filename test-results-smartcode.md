- 输入内容正常显示

#### ✅ 重新锁定功能
- 点击锁定按钮后，输入框恢复为禁用状态

## 测试总结

### ✅ 所有测试通过

1. **组件集成**：所有6个创建表单都已成功集成SmartCodeInput组件
2. **锁定状态**：默认锁定状态下，所有输入框都正确显示下一个将要生成的编号
3. **解锁功能**：所有表单的解锁按钮都能正常工作，点击后输入框变为可编辑状态
4. **占位符显示**：所有输入框都正确显示编号规则示例（如"PRD0001, PRD0002, PRD0003 ..."）
5. **手动输入**：解锁后可以正常手动输入自定义编号
6. **重新锁定**：重新锁定后，输入框恢复为禁用状态并显示自动生成的编号

### 编号规则验证

- **产品SKU**：PRD0007（前缀PRD + 4位数字）
- **报价单编号**：QUO格式（前缀QUO + 4位数字）
- **订单编号**：ORD00004（前缀ORD + 5位数字）
- **供应商编号**：SUP0010（前缀SUP + 4位数字）
- **客户编号**：CUS0003（前缀CUS + 4位数字）
- **批次编号**：variant格式

### 发现的问题
无

### 后续建议

1. **表单提交测试**：测试提交表单时，锁定状态下是否正确使用自动生成的编号，解锁状态下是否正确使用手动输入的编号
2. **SKU规则变化测试**：修改SKU规则设置（如修改前缀或位数），验证占位符和下一个编号是否实时更新
3. **错误处理测试**：测试手动输入重复编号时的错误提示
4. **批次编号特殊逻辑测试**：批次编号可能有特殊的格式要求（如产品SKU-V001），需要进一步测试
