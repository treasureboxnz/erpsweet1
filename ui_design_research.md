# 现代化B2B CRM客户详情页面UI设计研究

## 研究来源
- Dribbble CRM设计作品集
- 现代SaaS CRM产品（HubSpot、Salesforce、Pipedrive等）

## 核心设计模式总结

### 1. 页面布局结构
**顶部区域（Header）**
- 客户公司Logo + 公司名称
- 状态标签（活跃/潜在/停用）
- 重要性标签（高/中/低）
- 主要操作按钮（编辑、删除、导出等）
- 返回按钮

**统计卡片区（Stats Cards）**
- 4-6个关键指标卡片横向排列
- 每个卡片包含：图标、标题、数值、趋势指标
- 使用不同颜色区分不同类型的指标

**标签页导航（Tab Navigation）**
- 清晰的标签页切换（概览/联系人/活动/订单/文档等）
- 标签页下方有分隔线
- 当前标签页有明显的视觉反馈（下划线或背景色）

**内容区域（Content Area）**
- 根据标签页显示不同内容
- 使用卡片式布局组织信息
- 每个卡片有标题、操作按钮、内容区

### 2. 联系人管理UI模式

**卡片式布局（推荐）**
- 每个联系人一张卡片
- 卡片包含：头像、姓名、职位、联系方式
- 卡片右上角有编辑/删除按钮（hover显示）
- 角色标签和重要性标签用不同颜色区分
- 网格布局（2-3列）或列表布局

**关键元素**
- 头像（圆形或方形）
- 姓名（大字体、加粗）
- 职位/角色标签（小标签）
- 联系方式图标（电话、邮箱、微信、LinkedIn等）
- 操作按钮（编辑、删除、发送邮件等）

### 3. 跟进记录/活动时间线UI模式

**时间线布局（推荐）**
- 垂直时间线，左侧显示时间
- 每条记录用卡片或列表项展示
- 不同类型的活动用不同图标和颜色标识
- 最新的记录在顶部

**记录卡片元素**
- 活动类型图标（电话、邮件、会议、拜访等）
- 标题/主题
- 内容摘要（可展开查看详情）
- 操作人和时间
- 操作按钮（编辑、删除、回复等）

### 4. 颜色和视觉设计

**现代化配色方案**
- 主色：蓝色系（#3B82F6、#2563EB）
- 成功/活跃：绿色系（#10B981、#059669）
- 警告/待处理：黄色/橙色系（#F59E0B、#F97316）
- 危险/停用：红色系（#EF4444、#DC2626）
- 中性：灰色系（#6B7280、#9CA3AF）

**卡片设计**
- 白色背景
- 轻微阴影（shadow-sm）
- 圆角（rounded-lg）
- 内边距充足（p-4 到 p-6）
- hover时阴影加深

**间距和排版**
- 使用8px网格系统
- 标题和内容之间有明确的层次
- 充足的留白，避免拥挤

### 5. 交互设计

**Hover效果**
- 卡片hover时阴影加深
- 操作按钮hover时显示
- 列表项hover时背景色变化

**加载状态**
- 使用骨架屏（Skeleton）而非loading spinner
- 乐观更新（Optimistic Update）提升体验

**操作反馈**
- 成功操作显示Toast提示
- 危险操作（删除）需要确认对话框
- 表单验证实时反馈

## 具体实施方案

### 客户详情页面重构
1. **顶部区域**
   - 左侧：返回按钮 + 公司Logo + 公司名称 + 状态标签
   - 右侧：编辑按钮

2. **统计卡片**
   - 订单数量、USD总额、RMB总额、平均订单金额
   - 4个卡片横向排列

3. **标签页导航**
   - 概览、联系人(数量)、跟进记录(数量)、订单历史(数量)
   - 使用shadcn/ui的Tabs组件

4. **联系人卡片**
   - 网格布局（2列）
   - 每个卡片显示头像、姓名、职位、联系方式
   - 右上角编辑/删除按钮

5. **跟进记录时间线**
   - 垂直时间线布局
   - 不同类型用不同图标和颜色
   - 每条记录可展开查看详情
